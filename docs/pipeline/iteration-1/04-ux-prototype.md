# UX Prototype: Tech Knowledge Base (Строгий Цифровой Сад)

## Допущения

1. **Прототип `Сайт3.html` -- эталон дизайна.** Все wireframe-макеты воспроизводят layout, типографику, интерактивность и визуальную иерархию прототипа один к одному. Отклонения документируются явно.
2. **MPA-модель (Astro SSG).** Fade-анимация при переходах из прототипа (SPA) не воспроизводится в v1. Навигация -- стандартные page transitions.
3. **Контент фиксирован: 3 записи.** Use-cases описывают поведение как для текущего объёма (3 записи, 8 тегов), так и для целевого (20+ записей).
4. **Язык интерфейса -- русский.** Все UI-тексты, метки, сообщения об ошибках -- на русском.
5. **v1 не включает:** полнотекстовый поиск, тёмную тему, дашборд-статистику (Chart.js), бэклог-виджет (lined-paper), виджет метрик (before/after).
6. **Правая колонка главной страницы в v1 пуста** -- блоки "Статистика" и "Бэклог" отложены до v2. В v1 используется упрощённый layout без правой колонки, либо правая колонка содержит placeholder-текст с описанием блога.

## Use Cases

### UC-1: Просмотр главной страницы (Overview)
- **Actor:** Читатель (посетитель сайта)
- **Precondition:** Пользователь открыл корневой URL сайта (`/`) или нажал "Обзор" в sidebar.
- **Main Flow:**
  1. Система загружает главную страницу с layout: sidebar слева + контентная область справа.
  2. Система отображает заголовок "ОБЗОР" крупным шрифтом по центру контентной области.
  3. Система отображает блок "Недавние записи" в виде bujo-box с инвертированным заголовком.
  4. Внутри блока -- список записей (до 10), отсортированных по дате (новые первые). Каждая строка: чёрный круглый маркер (12px), заголовок записи, дата в моноширинном шрифте.
  5. Пользователь наводит курсор на строку записи.
  6. Система показывает hover-эффект: маркер увеличивается (scale-150), заголовок подчёркивается (decoration-2, underline-offset-4).
  7. Пользователь кликает на строку записи.
  8. Система переходит на детальную страницу записи (`/notes/[slug]/`).
- **Alternative Flows:**
  - 4a. Если записей нет (теоретический сценарий при пустой базе): система отображает текст "Записей пока нет. Скоро здесь появится первый материал." в моноширинном шрифте.
  - 7a. Пользователь кликает на дату записи -- система также переходит на детальную страницу (вся строка -- кликабельная область).
- **Postcondition:** Пользователь находится на детальной странице выбранной записи, либо остаётся на главной странице (если не кликал).
- **Related Stories:** Story 3.3

### UC-2: Просмотр списка всех записей
- **Actor:** Читатель
- **Precondition:** Пользователь нажал "Все записи" в sidebar или перешёл по URL `/notes/`.
- **Main Flow:**
  1. Система загружает страницу списка всех записей.
  2. Система отображает заголовок "ВСЕ ЗАПИСИ" по центру с нижним бордером (border-b-4 border-black).
  3. Система отображает вертикальный список карточек NoteCard, отсортированных по дате (новые первые).
  4. Каждая карточка содержит: блок даты слева (число в крупном шрифте + год моноширинным шрифтом), заголовок справа (font-sans, font-bold, text-2xl), теги в виде бейджей под заголовком.
  5. Пользователь наводит курсор на карточку.
  6. Система инвертирует цвета всей карточки: чёрный фон, белый текст, бордеры карточки меняются на белые. Заголовок получает подчёркивание.
  7. Пользователь кликает на карточку.
  8. Система переходит на детальную страницу записи (`/notes/[slug]/`).
- **Alternative Flows:**
  - 4a. На мобильном устройстве (< 768px): карточка перестраивается в вертикальный layout (flex-col) -- дата сверху, контент снизу.
- **Postcondition:** Пользователь находится на детальной странице выбранной записи.
- **Related Stories:** Story 3.4, Story 2.3

### UC-3: Фильтрация записей по тегу
- **Actor:** Читатель
- **Precondition:** Пользователь видит облако тегов в sidebar, или теги в карточке записи, или теги на детальной странице.
- **Main Flow:**
  1. Пользователь кликает на тег (в sidebar, в карточке NoteCard, или на детальной странице записи).
  2. Система переходит на страницу фильтрации `/tags/[tag-slug]/`.
  3. Система отображает заголовок "ТЕМА: #<Тег>" (оригинальное кириллическое написание тега) по центру с нижним бордером.
  4. Система отображает список карточек NoteCard, содержащих данный тег, отсортированных по дате.
  5. Карточки визуально и поведенчески идентичны карточкам на странице "Все записи" (UC-2, шаги 4-8).
- **Alternative Flows:**
  - 4a. Если ни одна запись не содержит данного тега (теоретически -- если тег удалён из frontmatter после генерации страницы): система отображает текст "Нет записей по теме #<Тег>".
  - 1a. В sidebar: hover на тег показывает инверсию (чёрный фон, белый текст, появляется бордер). Клик выполняет переход.
- **Postcondition:** Пользователь видит отфильтрованный список записей по выбранному тегу.
- **Related Stories:** Story 3.5

### UC-4: Чтение записи (детальная страница)
- **Actor:** Читатель
- **Precondition:** Пользователь кликнул на запись из любого списка (главная, все записи, фильтр по тегу) или перешёл по прямой ссылке `/notes/[slug]/`.
- **Main Flow:**
  1. Система загружает детальную страницу записи.
  2. В верхнем левом углу контентной области отображается кнопка "НАЗАД" (моноширинный шрифт, border-2, иконка arrow-left слева от текста).
  3. Ниже -- bujo-box с записью, разделённый на header и body.
  4. Header (чёрный фон, белый текст, по центру):
     - Метка "ЗАПИСЬ // DD.MM.YYYY" (моноширинный шрифт, tracking-[0.3em], uppercase, с нижней разделительной линией border-white/30).
     - Заголовок записи (font-display, font-black, uppercase, text-3xl на мобильных / text-5xl на десктопе).
     - Теги записи (font-mono, border белый, фон белый, текст чёрный, font-bold) -- размещены по центру горизонтально с gap-3.
  5. Body (белый фон, padding 32px на мобильных / 48px на десктопе):
     - Отрендеренный Markdown/MDX-контент с prose-стилизацией.
     - Блоки кода с синтаксической подсветкой (Shiki, монохромная тема).
     - Кастомные компоненты (bujo-box внутри контента, чек-листы, grid-сетки) -- если запись использует MDX.
  6. Пользователь прокручивает контент вертикально для чтения.
  7. По завершении чтения пользователь нажимает кнопку "НАЗАД".
  8. Система переходит на главную страницу (`/`).
- **Alternative Flows:**
  - 2a. Hover на кнопку "НАЗАД": инверсия цветов (чёрный фон, белый текст).
  - 7a. Пользователь может использовать навигацию sidebar вместо кнопки "НАЗАД" -- перейти на любую другую страницу.
  - 1a. Если slug не найден (запись удалена/переименована): система отображает страницу 404 (UC-6).
- **Postcondition:** Пользователь прочитал запись и вернулся к навигации.
- **Related Stories:** Story 3.6, Story 6.1, Story 6.2

### UC-5: Навигация через sidebar
- **Actor:** Читатель
- **Precondition:** Сайт загружен; sidebar видим (десктоп) или доступен (мобильный).
- **Main Flow:**
  1. Система отображает sidebar с фиксированной шириной (w-72) на десктопе, разделённый на секции:
     - Заголовок "ИНДЕКС" (font-display, font-black, uppercase, text-4xl, по центру) + подзаголовок "ТЕХНИЧЕСКАЯ БАЗА ЗНАНИЙ" (font-mono, 10px, tracking-[0.2em], uppercase).
     - Блок "Навигация" (bujo-box с инвертированным заголовком): два пункта -- "Обзор" (номер 01) и "Все записи" (номер 02).
     - Блок "Темы" (bujo-box с обычным заголовком): облако тегов -- каждый тег как отдельная кнопка в вертикальном списке с префиксом "#".
     - Подвал "ПУБЛИЧНЫЙ РЕСУРС" (font-mono, 10px, uppercase, tracking-widest, border-t-2).
  2. Пользователь наводит курсор на пункт навигации.
  3. Система делает текст жирным (hover:font-bold).
  4. Пользователь кликает на "Обзор".
  5. Система переходит на главную страницу (`/`).
  6. Пользователь кликает на "Все записи".
  7. Система переходит на страницу списка записей (`/notes/`).
  8. Пользователь кликает на тег в облаке тегов.
  9. Система переходит на страницу фильтрации по тегу (`/tags/[tag-slug]/`).
- **Alternative Flows:**
  - 2a. Hover на тег в облаке: чёрный фон, белый текст, появляется border-black.
  - 1a. На мобильном устройстве (< 768px): sidebar адаптируется -- отображается как сворачиваемая панель сверху (details/summary) или горизонтальное меню.
- **Postcondition:** Пользователь перешёл на выбранную страницу.
- **Related Stories:** Story 2.2, Story 4.1, TECH-7

### UC-6: Переход на несуществующую страницу (404)
- **Actor:** Читатель
- **Precondition:** Пользователь ввёл URL, которому не соответствует ни одна страница сайта, или перешёл по устаревшей ссылке.
- **Main Flow:**
  1. Система определяет, что запрошенный маршрут не существует.
  2. Система отображает страницу 404 с использованием BaseLayout (sidebar остаётся видимым).
  3. В контентной области отображается:
     - Заголовок "404" (font-display, font-black, очень крупный размер, по центру).
     - Подзаголовок "СТРАНИЦА НЕ НАЙДЕНА" (font-display, uppercase, размер меньше).
     - Пояснительный текст: "Этой страницы не существует. Возможно, она была перемещена или удалена." (font-sans).
     - Кнопка "ВЕРНУТЬСЯ НА ГЛАВНУЮ" (bujo-стиль: border-2, font-mono, font-bold, hover:bg-black hover:text-white).
  4. Пользователь нажимает "ВЕРНУТЬСЯ НА ГЛАВНУЮ".
  5. Система переходит на главную страницу (`/`).
- **Alternative Flows:**
  - 4a. Пользователь использует sidebar для навигации вместо кнопки.
- **Postcondition:** Пользователь вернулся к рабочему маршруту сайта.
- **Related Stories:** Story 7.2

### UC-7: Подписка на RSS-ленту
- **Actor:** Читатель (технический специалист, использующий RSS-ридер)
- **Precondition:** Пользователь находится на любой странице сайта и хочет подписаться на обновления.
- **Main Flow:**
  1. Пользователь видит иконку RSS (или текстовую ссылку "RSS") в sidebar (подвал или блок навигации).
  2. Пользователь кликает на ссылку.
  3. Браузер открывает RSS-ленту (`/rss.xml`) или пользователь копирует URL ленты.
  4. Пользователь добавляет URL в свой RSS-ридер.
- **Alternative Flows:**
  - 1a. Если иконка RSS размещена в подвале sidebar: она расположена рядом с надписью "Публичный Ресурс".
- **Postcondition:** Пользователь подписан на обновления блога через RSS.
- **Related Stories:** Story 7.1

### UC-8: Индексация страницы поисковой системой
- **Actor:** Поисковый бот (Googlebot, Yandexbot)
- **Precondition:** Сайт опубликован на GitHub Pages и доступен по HTTPS.
- **Main Flow:**
  1. Бот запрашивает `/robots.txt`.
  2. Система возвращает содержимое robots.txt с разрешением индексации и ссылкой на sitemap.
  3. Бот запрашивает `/sitemap-index.xml`.
  4. Система возвращает XML-карту сайта со всеми страницами.
  5. Бот запрашивает каждую страницу из sitemap.
  6. Система возвращает HTML с корректными мета-тегами: `<title>`, `description`, `og:title`, `og:description`, `og:url`, `og:type`, `lang="ru"`.
  7. HTML содержит семантическую разметку: `<article>`, `<nav>`, `<main>`, `<aside>`, `<header>`, `<time>`.
- **Alternative Flows:**
  - 5a. Бот запрашивает страницу, удалённую после последней индексации: система возвращает 404.
- **Postcondition:** Страницы блога проиндексированы поисковой системой и появляются в результатах поиска.
- **Related Stories:** Story 5.1, Story 5.2

### UC-9: Просмотр сайта на мобильном устройстве
- **Actor:** Читатель с мобильного устройства (ширина экрана < 768px)
- **Precondition:** Пользователь открывает сайт на смартфоне или планшете.
- **Main Flow:**
  1. Система загружает страницу. Sidebar не отображается как боковая колонка.
  2. Вместо sidebar отображается сворачиваемая панель (details/summary) или горизонтальная панель с бургер-кнопкой сверху.
  3. По умолчанию навигация свёрнута -- виден только заголовок "ИНДЕКС" и кнопка раскрытия.
  4. Пользователь нажимает на кнопку раскрытия.
  5. Система раскрывает навигационную панель: пункты меню "Обзор" / "Все записи" и облако тегов.
  6. Пользователь выбирает пункт меню.
  7. Система переходит на выбранную страницу. Навигация автоматически сворачивается.
  8. Контент страницы отображается в одну колонку на всю ширину экрана.
  9. Карточки записей отображаются в вертикальном layout (дата сверху, заголовок снизу).
  10. Блоки кода имеют горизонтальную прокрутку (overflow-x-auto).
- **Alternative Flows:**
  - 2a. Альтернативный вариант: sidebar отображается как горизонтальная полоска сверху с ключевыми элементами навигации без сворачивания.
  - 10a. Длинные строки в таблицах или блоках кода: горизонтальная прокрутка без обрезки контента.
- **Postcondition:** Пользователь комфортно просматривает контент на мобильном устройстве.
- **Related Stories:** Story 4.1, TECH-7

## Content Strategy

### Tone of Voice

Строгий, утилитарный, технический. Минимум слов -- максимум смысла. Никаких приветствий, эмодзи, маркетинговых формулировок. Тон напоминает инженерный журнал: сухие факты, ясные формулировки, профессиональная терминология без упрощений. Допускается использование технического сленга и англицизмов (ETL, DAG, LLM, pipeline), принятых в профессиональной среде.

Регистр заголовков -- UPPERCASE (все заглавные). Это часть визуального языка дизайн-системы, а не крик. Основной текст -- обычный регистр, стандартная пунктуация.

### Microcopy Guidelines

- **Error messages (ошибки):** Паттерн: факт + возможная причина + действие. Без восклицательных знаков, без слов "Ой" или "Упс". Пример: "Этой страницы не существует. Возможно, она была перемещена или удалена."
- **Success messages (подтверждения):** Не применимо для v1 -- сайт не имеет пользовательского ввода и действий, требующих подтверждения.
- **Loading states (загрузка):** Не применимо для статического сайта -- страницы загружаются мгновенно. Если потребуется (v2, поиск): "Поиск..." (font-mono, text-sm).
- **Навигационные метки:** Всегда uppercase для заголовков блоков. Обычный регистр для пунктов меню. Моноширинный шрифт для всех метаданных.
- **Даты:** Формат DD.MM для коротких дат (в списках) и DD.MM.YYYY для полных (на детальной странице). Год отдельно моноширинным шрифтом (text-xs, uppercase).
- **Теги:** Префикс "#" используется в sidebar и на детальной странице. В карточках списка -- без "#" (визуально чище). Регистр тегов -- как в frontmatter (смешанный: "1C-Analytics", "Оптимизация").

### Responsive Behavior

#### Desktop (>= 1024px / lg breakpoint)
- **Layout:** Двухколоночный -- sidebar (w-72 = 288px фиксированная) + main content area (flex-1)
- **Sidebar:** Фиксированная колонка слева, полная высота (h-full), border-r-2, overflow-y-auto внутри
- **Main:** flex-1, overflow-y-auto, горизонтальные padding px-20 (80px)
- **Grid на главной:** 3 колонки (md:grid-cols-3), записи занимают col-span-2
- **NoteCard:** Горизонтальный layout (flex-row) -- дата слева (md:w-32), контент справа
- **Детальная страница:** max-w-3xl (768px), header padding p-12

#### Tablet (768px -- 1024px / md breakpoint)
- **Layout:** Аналогичен десктопу -- sidebar + main. Sidebar по-прежнему w-72.
- **Main:** Горизонтальные padding px-12 (48px)
- **Grid на главной:** 3 колонки (md:grid-cols-3)
- **NoteCard:** Горизонтальный layout (flex-row)
- **Детальная страница:** max-w-3xl, header padding p-8

#### Mobile (< 768px)
- **Layout:** Одноколоночный. Sidebar сверху в виде сворачиваемой панели.
- **Sidebar:** Горизонтальная панель сверху. По умолчанию свёрнута -- видна только полоска с заголовком "ИНДЕКС" и кнопкой раскрытия. При раскрытии: блоки "Навигация" и "Темы" отображаются вертикально.
- **Main:** Полная ширина, горизонтальные padding px-6 (24px)
- **Grid на главной:** 1 колонка (grid-cols-1)
- **NoteCard:** Вертикальный layout (flex-col) -- дата сверху (border-b-2 вместо border-r-2), контент снизу
- **Детальная страница:** max-w-full, header padding p-8, заголовок text-3xl (вместо text-5xl)
- **Блоки кода:** overflow-x-auto для горизонтальной прокрутки
- **Touch targets:** Все кнопки, ссылки, теги -- минимум 44x44px
- **Шрифты:** Основной текст не менее 14px для читабельности
