# Product Backlog & Design Concept: Tech Knowledge Base (Строгий Цифровой Сад)

## Допущения

1. **SSG-фреймворк — Astro.** Выбран как оптимальный вариант: нативная поддержка Markdown/MDX, компонентная архитектура, интеграция с Tailwind CSS, поддержка View Transitions API, нулевой JS по умолчанию. Альтернативы (11ty, Hugo) рассматриваются, но бэклог ориентирован на Astro.
2. **Стратегия slug-генерации — транслитерация.** Кириллические теги и заголовки будут транслитерированы в латиницу для URL (`Оптимизация` -> `optimizaciya`). Оригинальные кириллические названия сохраняются в frontmatter и отображаются в UI.
3. **Markdown + кастомные компоненты (MDX или Astro-компоненты).** Для воспроизведения bujo-box, чек-листов и сеток из прототипа будут созданы переиспользуемые компоненты, вызываемые из контента.
4. **Хостинг — GitHub Pages** как первый выбор, с возможностью переключения на Netlify/Vercel.
5. **Шрифты будут самохостированы** (через `@fontsource` или локальные файлы) для устранения зависимости от Google Fonts CDN.
6. **Три существующие записи из прототипа — весь контент для v1.** Новый контент не создаётся в рамках v1.

### Epic 1: Инициализация проекта и инфраструктура

  - [ ] Подключены Lucide Icons через npm-пакет `lucide-astro` или аналог

- **Estimated complexity:** M

## Design Concept

### Visual Language

#### Цветовая палитра
- **Primary:** `#000000` (чёрный) — рамки, текст, заголовки, hover-состояния, заливки акцентных блоков
- **Secondary:** `#FFFFFF` (белый) — фон контента, фон карточек (`rgba(255,255,255,0.95)`)
- **Accent:** `#D1D5DB` (серый) — точки фоновой сетки, линии линовки (`.lined-paper`), вспомогательные разделители
- **Нет промежуточных цветов.** Палитра бинарная. Серый используется исключительно для фоновых паттернов, не для текста и не для UI-элементов.

#### Типографика
| Роль | Шрифт | Применение | Стиль |
|------|-------|------------|-------|
| Display / Заголовки | **Unbounded** | Заголовки страниц, заголовки блоков, навигационные секции | `uppercase`, `font-black` / `font-bold`, `tracking-tight` / `tracking-wider` |
| Body / Основной текст | **Geist** | Контент записей, описания, списки | `font-sans`, `font-medium`, обычный регистр |
| Mono / Технический текст | **Geist Mono** | Даты, теги, метаданные, код, навигационные номера, подвал | `font-mono`, `text-xs` / `text-sm`, часто `uppercase`, `tracking-widest` |

#### Принципы дизайна

1. **Тотальный монохром.** Только чёрный и белый. Никаких цветовых акцентов, градиентов, теней. Контраст создаётся толщиной линий, инверсией и типографической иерархией.

2. **Инженерная геометрия.** Все элементы — прямоугольные. Border-radius = 0 (единственное исключение — маркеры в списке записей на главной, но это осознанный выбор прототипа). Рамки — 2px solid. Визуальная привязка к точечной сетке 24px.

3. **Информационная плотность.** Минимум декоративных элементов. Каждый пиксель несёт функцию. Whitespace используется стратегически для создания иерархии, а не для «воздуха».

4. **Инверсия как интерактивность.** Единственный способ показать hover/active — инверсия цветов (чёрный фон + белый текст) или подчёркивание (decoration-2, underline-offset-4). Нет fade, glow, transform (кроме scale маркеров).

5. **Шрифт как UI-элемент.** Три шрифта образуют визуальную иерархию: Unbounded = «заголовок/секция», Geist = «читай это», Geist Mono = «метаданные/код».

### Key Screens / Views

#### 1. Главная страница (Overview / Dashboard)
- **Назначение:** Точка входа. Показывает, что на сайте есть, и даёт быстрый доступ к записям.
- **Ключевые элементы:**
  - Заголовок "ОБЗОР" — крупный, по центру (font-display, text-7xl на десктопе)
  - Сетка 2 колонки + 1 колонка (md:grid-cols-3)
  - Левая часть (col-span-2): блок "Недавние записи" в bujo-box. Список: маркер + заголовок + дата. Кликабельные строки.
  - Правая часть (col-span-1): зарезервирована для будущих виджетов (v2: статистика, бэклог). В v1 может быть пустой или содержать краткое описание блога.
- **User flow:** Загрузка -> сканирование списка записей -> клик на заголовок -> переход на детальную страницу.

#### 2. Список записей (All Logs)
- **Назначение:** Полный каталог всех записей блога.
- **Ключевые элементы:**
  - Заголовок "ВСЕ ЗАПИСИ" — по центру, с нижним бордером (border-b-4 border-black)
  - Вертикальный список карточек NoteCard
  - Каждая карточка: горизонтальная компоновка (flex-row на десктопе) — дата слева (крупный шрифт + год), заголовок + теги справа
  - Hover: инверсия всей карточки
- **User flow:** Навигация из sidebar -> просмотр списка -> клик на карточку -> детальная страница.

#### 3. Записи по тегу (Topic Filter)
- **Назначение:** Подмножество записей, отфильтрованных по выбранному тегу.
- **Ключевые элементы:**
  - Заголовок "ТЕМА: #<ТЕГ>" — аналогичный стиль
  - Тот же список NoteCard, но отфильтрованный
  - При отсутствии записей — сообщение "Нет записей с этим тегом"
- **User flow:** Клик на тег (в sidebar, в карточке, на детальной странице) -> просмотр отфильтрованного списка.

#### 4. Детальная страница записи (Log Entry)
- **Назначение:** Полный контент одной записи. Основная ценность сайта.
- **Ключевые элементы:**
  - Кнопка "НАЗАД" (верхний левый угол, bujo-стиль с иконкой arrow-left)
  - Header записи в bujo-box: чёрный фон, белый текст. Содержит: метку "Запись // DD.MM.YYYY", заголовок (display font, uppercase), теги (белые бейджи на чёрном)
  - Body записи: белый фон, prose-стилизация. Markdown-контент с подсветкой кода.
  - Max-width: 768px (max-w-3xl) для комфортного чтения.
- **User flow:** Переход с любого списка -> чтение контента -> клик "Назад" -> возврат на главную.

#### 5. Страница 404
- **Назначение:** Информирование пользователя о несуществующей странице.
- **Ключевые элементы:**
  - Заголовок "404" (font-display, очень крупный, по центру)
  - Подзаголовок "Страница не найдена"
  - Кнопка "Вернуться на главную" в bujo-стиле
  - Sidebar остаётся видимым (пользователь может навигировать оттуда)

### Interaction Patterns

#### Навигация
- **Паттерн:** Фиксированный sidebar (desktop) + контентная область с вертикальным скроллом.
- **Sidebar:** Всегда видим на десктопе (w-72, border-r-2). На мобильных — адаптация (горизонтальное меню или бургер).
- **Переходы:** Стандартные переходы между страницами (MPA). Без SPA-анимаций в v1. В будущем (v2) можно добавить View Transitions API для fade-эффекта.
- **Навигационные пункты:** Два основных ("Обзор", "Все записи") + динамическое облако тегов. Нумерация пунктов (01, 02) — декоративная деталь из прототипа.

#### Ввод данных
- **Отсутствует.** Сайт не предполагает пользовательского ввода. Контент создаётся автором через Markdown-файлы и Git. Формы, комментарии, поиск — не в scope v1.

#### Обратная связь (системное состояние)
- **Hover-состояния:** Инверсия цветов (bg-black text-white) для карточек и кнопок. Подчёркивание (decoration-2 underline-offset-4) для текстовых ссылок. Scale-эффект (scale-150) для маркеров.
- **Активная страница:** В sidebar можно визуально выделить текущий пункт навигации (font-bold или инверсия).
- **Загрузка:** Не требуется для статического сайта. Все страницы загружаются мгновенно.
- **Ошибки:** Единственный сценарий — 404 (несуществующая страница). Обрабатывается кастомной страницей.

### Design Constraints

#### Технические ограничения
- **Статическая генерация (SSG):** Все страницы генерируются на этапе сборки. Нет серверного рендеринга, нет динамических API. Это исключает любые интерактивные элементы, требующие бэкенда (поиск, комментарии, аналитика в реальном времени).
- **Нулевой клиентский JavaScript (цель):** Astro по умолчанию не отправляет JS на клиент. Все hover-эффекты реализуются через CSS. Если потребуется JS (например, для бургер-меню на мобильных), использовать минимальный inline-скрипт.
- **Tailwind CSS через процесс сборки:** Только используемые классы попадают в итоговый CSS. Произвольные классы из прототипа (`text-[10px]`, `tracking-[0.2em]`, `tracking-[0.3em]`) должны быть перенесены в конфигурацию или использоваться как arbitrary values.
- **Кириллица в slug'ах:** Требуется транслитерация для URL. Необходимо выбрать и зафиксировать библиотеку/функцию транслитерации для единообразия.
- **GitHub Pages ограничения:** Только статические файлы, нет серверных редиректов. 404-страница должна быть файлом `404.html` в корне.

#### Требования доступности (Accessibility)
- **Контрастность:** Монохромная палитра (#000 на #FFF) обеспечивает максимальный контраст (21:1). Не требует дополнительной работы.
- **Семантическая разметка:** Использовать `<nav>`, `<main>`, `<article>`, `<aside>`, `<header>`, `<time>` для корректной работы screen readers.
- **Клавиатурная навигация:** Все интерактивные элементы (ссылки, кнопки) должны быть фокусируемыми. Focus-ring стилизован в соответствии с дизайн-системой (outline 2px solid black, offset 2px).
- **Alt-тексты:** Если в контенте появятся изображения (v2), обязательно указывать alt.
- **Touch targets:** На мобильных минимальный размер интерактивных элементов — 44x44px (стандарт WCAG).
- **Языковой атрибут:** `<html lang="ru">` для корректной работы синтезаторов речи.
