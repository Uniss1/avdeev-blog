# Технические Спецификации: Tech Knowledge Base (Строгий Цифровой Сад)

> **Контекст:** Статический сайт на Astro SSG, деплоится на GitHub Pages. Нет серверного бэкенда, REST API, базы данных. "API" в данном документе означает маршруты страниц, схемы Content Collections и потоки данных на этапе сборки.

> **Источники:** 03-architecture.md, 04-ux-prototype.md, 05-specifications.md, Сайт3.html (HTML-прототип).

> **Тестовое окружение:** Тесты, отмеченные как `unit`, выполняются через Vitest (`npm run test`). Тесты `e2e` выполняются через Playwright (`npm run test:e2e`). Тесты `build` проверяют артефакты сборки (`npm run build`). Тесты `lint` используют TypeScript-компилятор и Zod-валидацию на этапе сборки.

> **Допущения документа:**
> 1. Тестового фреймворка в проекте ещё нет — тест-кейсы написаны как спецификация для реализации. Рекомендуемый стек: Vitest + Playwright.
> 2. v1 содержит ровно 3 записи, 8 уникальных тегов (1C-Analytics, SQL, Оптимизация, AI-Агенты, LLM, Python, Пайплайны-данных, ETL, Airflow — итого 9, один тег из каждой записи пересекается в UI-прототипе).
> 3. Аналитика (Plausible, Яндекс Метрика) не входит в scope v1 — тест-кейсы для неё описаны как «отложенные» и помечены `[v1.1]`.
> 4. Slug записей определяется именем файла. Slug тегов генерируется утилитой `transliterate()`.
> 5. Базовый URL при деплое: `https://<username>.github.io/<repo>/`. Base path настраивается в `astro.config.mjs`.

## Тест-кейсы

### TC-1: Транслитерация кириллических символов
- **Связан с:** FR-20, TECH-1
- **Предусловия:** Файл `src/utils/transliterate.ts` создан и экспортирует функцию `transliterate(input: string): string`.
- **Шаги:**
  1. Вызвать `transliterate("Оптимизация")`.
  2. Вызвать `transliterate("AI-Агенты")`.
  3. Вызвать `transliterate("Пайплайны-данных")`.
  4. Вызвать `transliterate("1C-Analytics")`.
  5. Вызвать `transliterate("BI Аналитика")`.
  6. Вызвать `transliterate("")`.
  7. Вызвать `transliterate("Ёж")`.
- **Ожидаемый результат:**
  1. `"optimizaciya"`
  2. `"ai-agenty"`
  3. `"pajplajny-dannyh"`
  4. `"1c-analytics"`
  5. `"bi-analitika"` (пробел → дефис)
  6. `""` (пустая строка на входе → пустая строка на выходе)
  7. `"yozh"` (ё → yo, Ё → Yo; проверка символа ё/Ё)
- **Правила трансформации:**
  - Все символы переводятся в нижний регистр.
  - Пробелы заменяются на `-`.
  - Не-латинские, не-цифровые символы (кроме уже-существующих дефисов) удаляются.
  - Последовательные дефисы сжимаются в один.
  - Ведущие и замыкающие дефисы удаляются.
- **Тип теста:** unit

- **Ожидаемый результат:** Функция корректно маппирует каждый символ кириллицы. Выходная строка соответствует regex `^[a-z0-9-]*$`.

#### Маршрут: `/tags/[tag]/`

- **Функция генерации маршрутов (build-time):**
  ```typescript
  export async function getStaticPaths() {
    const notes = await getCollection('notes');
    const allTags = [...new Set(notes.flatMap(n => n.data.tags))];
    return allTags.map(tag => {
      const tagSlug = transliterate(tag);
      const filteredNotes = notes
        .filter(n => n.data.tags.includes(tag))
        .sort((a, b) => b.data.date.getTime() - a.data.date.getTime());
      return {
        params: { tag: tagSlug },
        props: { tagName: tag, tagSlug, notes: filteredNotes }
      };
    });
  }
  ```

### Схема Content Collection (`notes`)

**Файл конфигурации:** `src/content/config.ts`

```typescript
import { defineCollection, z } from 'astro:content';

const notesCollection = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string().min(1).max(200),
    date: z.coerce.date(),
    category: z.enum(['BI', 'AI', 'Инженерия']).or(z.string().min(1)),
    tags: z.array(z.string().min(1)).min(1).max(10),
    type: z.enum(['Техническое', 'Эксперимент', 'Лучшая практика']).or(z.string().min(1)),
    description: z.string().max(160).optional(),
    metrics: z.object({
      label: z.string(),
      before: z.number().nullable().optional(),
      after: z.number().nullable().optional(),
      value: z.number().nullable().optional(),
    }).nullable().optional(),
  }),
});

export const collections = {
  notes: notesCollection,
};
```

**Правила валидации полей:**

| Поле | Тип | Обязательность | Ограничения | Примечание |
|------|-----|----------------|-------------|------------|
| `title` | string | Обязательное | min 1, max 200 символов | Кириллица допустима |
| `date` | Date | Обязательное | Валидная дата, `z.coerce.date()` преобразует ISO-строку | Формат в файле: YYYY-MM-DD |
| `category` | string | Обязательное | Рекомендованные значения: "BI", "AI", "Инженерия"; допустима произвольная строка | Строка min 1 символ |
| `tags` | string[] | Обязательное | Массив строк, min 1 тег, max 10 тегов | Каждый тег min 1 символ |
| `type` | string | Обязательное | Рекомендованные: "Техническое", "Эксперимент", "Лучшая практика" | Строка min 1 символ |
| `description` | string | Опциональное | max 160 символов (лимит meta description) | Используется в SEO |
| `metrics` | object \| null | Опциональное | Поля: label (string), before (number \| null), after (number \| null), value (number \| null) | Зарезервировано для v2 |

**Автоматически добавляемые поля (Astro):**

| Поле | Тип | Источник | Примечание |
|------|-----|----------|------------|
| `slug` | string | Имя файла без расширения | Пример: файл `haki-dlya-optimizacii.md` → slug `haki-dlya-optimizacii` |
| `body` | string | Тело Markdown-файла (raw) | Доступен в компонентах страниц |
| `render()` | function | Astro | Возвращает `{ Content }` — скомпилированный компонент контента |

**Соглашение именования файлов:**

- Формат: `[транслитерированный-заголовок-записи].md` или `.mdx`
- Только строчные латинские буквы, цифры, дефисы
- Без пробелов и спецсимволов
- Примеры:
  - `haki-dlya-optimizacii-v-1c-analitike.md`
  - `lokalnye-llm-agenty-dlya-analiza-dokumentov.md`
  - `patterny-pipelinov-v-airflow.md`

## Требования к внешним интеграциям

### Интеграция: GitHub Pages (хостинг)

- **Назначение:** Хостинг статического сайта; CDN для раздачи файлов пользователям.
- **Версия:** Текущая (управляется GitHub, обновляется автоматически).
- **Аутентификация:** Не требуется для публичных репозиториев. Деплой через `GITHUB_TOKEN` (встроенный токен GitHub Actions).
- **Конфигурация:**

| Параметр | Значение |
|----------|----------|
| Source branch | `gh-pages` (или директория `/(root)` на branch `gh-pages`; управляется `withastro/action`) |
| Custom domain | Нет (v1). Опционально — CNAME в v2 |
| HTTPS | Включён автоматически (Let's Encrypt) |
| CDN | Fastly (глобальный) |

- **Лимиты и ограничения:**

| Параметр | Лимит |
|----------|-------|
| Размер репозитория | 5 GB (soft limit) |
| Размер сайта (dist/) | 1 GB |
| Пропускная способность | 100 GB/месяц (soft limit) |
| Время сборки (GitHub Actions) | 6 часов на job; 2000 минут/месяц для публичных репо |
| Кастомные HTTP-заголовки | Не поддерживаются (ограничение платформы) |

- **Обработка ошибок:**
  - 404: GitHub Pages автоматически отдаёт `dist/404.html` при несуществующем URL.
  - Превышение лимита пропускной способности: GitHub отправляет уведомление на email; сайт временно недоступен.
- **Фолбэк при недоступности:** GitHub Pages SLA не гарантирует uptime (бесплатный тир). Резервный вариант для v2: Cloudflare Pages (бесплатный тир, аналогичная функциональность).

---

### Интеграция: GitHub Actions (CI/CD)

- **Назначение:** Автоматическая сборка и деплой при push в ветку `main`.
- **Версия Actions:**
  - `actions/checkout@v4`
  - `actions/setup-node@v4` (Node.js 20)
  - `withastro/action@v3` (Astro deploy action)
- **Аутентификация:** `GITHUB_TOKEN` (встроенный, не требует настройки для публичных репозиториев).
- **Workflow файл:** `.github/workflows/deploy.yml`

**Конфигурация workflow:**
```yaml
name: Deploy to GitHub Pages
on:
  push:
    branches: [main]
permissions:
  contents: read
  pages: write
  id-token: write
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - run: npm run build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: dist/
  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/deploy-pages@v4
        id: deployment
```

- **Используемые переменные окружения:**

| Переменная | Источник | Назначение |
|------------|----------|------------|
| `GITHUB_TOKEN` | Встроенный | Аутентификация деплоя |
| (нет) | — | Нет дополнительных secrets в v1 |

- **Обработка ошибок:**
  - Сборка с ошибкой: workflow помечается как `failed`; деплой не выполняется; автор получает email-уведомление от GitHub.
  - Ошибка деплоя: job `deploy` помечается как `failed`; предыдущая версия сайта остаётся доступной на GitHub Pages.
- **Фолбэк:** Ручной деплой через `npm run build` + загрузка `dist/` в ветку `gh-pages`.

### Интеграция: Яндекс Метрика `[v1.1 — не в scope v1]`

- **Назначение:** Поведенческая аналитика (Вебвизор, тепловые карты, карта скроллов); интеграция с Яндекс.Вебмастером для индексации.
- **Версия:** tag.js (актуальная версия, загружается с `mc.yandex.ru`).
- **Аутентификация:** Номер счётчика (идентификатор, не секрет). Получается в Яндекс Метрике при создании счётчика.
- **Подключение:** Код счётчика в `BaseLayout.astro` перед `</head>`:
  ```html
  <script type="text/javascript">
    (function(m,e,t,r,i,k,a){...})(window, document, "script",
      "https://mc.yandex.ru/metrika/tag.js", "ym");
    ym(XXXXXXXX, "init", {
      clickmap: true, trackLinks: true,
      accurateTrackBounce: true, webvisor: true
    });
  </script>
  <noscript><div><img src="https://mc.yandex.ru/watch/XXXXXXXX"
    style="position:absolute; left:-9999px;" alt="" /></div></noscript>
  ```
  - Заменить `XXXXXXXX` на реальный номер счётчика.

- **Используемые домены и ресурсы:**

| Ресурс | Назначение | Размер |
|--------|-----------|--------|
| `https://mc.yandex.ru/metrika/tag.js` | Основной скрипт Метрики | ~35 KB |
| `https://mc.yandex.ru/watch/XXXXXXXX` | noscript-пиксель | ~0.5 KB |
| `https://yastatic.net/...` | Статические ресурсы Вебвизора | ~150 KB (lazy) |
| `https://mc.yandex.com/...` | Альтернативный домен | — |

- **Маппинг целей:**

| Наше событие (GA-план) | Идентификатор цели в Метрике | Вызов |
|------------------------|------------------------------|-------|
| `note_view` | `note-view` | `ym(ID, 'reachGoal', 'note-view', {slug, category})` |
| `tag_filter` | `tag-filter` | `ym(ID, 'reachGoal', 'tag-filter', {tag})` |
| `outbound_link` | `outbound-link` | `ym(ID, 'reachGoal', 'outbound-link', {url})` |
| `rss_subscribe` | `rss-subscribe` | `ym(ID, 'reachGoal', 'rss-subscribe')` |
| `404_hit` | `404-hit` | `ym(ID, 'reachGoal', '404-hit', {path})` |
| `scroll_depth` | `scroll-25` / `scroll-50` / `scroll-75` / `scroll-100` | `ym(ID, 'reachGoal', 'scroll-75', {slug})` |
| `back_button` | `back-button` | `ym(ID, 'reachGoal', 'back-button')` |
| `sidebar_nav` | `sidebar-nav` | `ym(ID, 'reachGoal', 'sidebar-nav', {nav_item})` |

- **Параметры визитов (для сегментации):**
  ```javascript
  // На детальной странице записи:
  ym(ID, 'params', {
    content: {
      category: note.data.category,
      type: note.data.type,
      tags: note.data.tags,
      reading_time: Math.ceil(wordCount / 200)  // слов / 200 слов в минуту
    }
  });
  ```

- **Обработка ошибок:**
  - Если `mc.yandex.ru` недоступен: `ym` не определена → события не отправляются → сайт работает штатно.
  - Защитная обёртка: `typeof ym !== 'undefined' && ym(...)`.
  - noscript-пиксель обеспечивает базовый учёт при отключённом JavaScript.

- **Фолбэк:** При недоступности сервиса сайт полностью функционален. Данные Вебвизора за период недоступности теряются.

- **Влияние на производительность:**
  - Скрипт tag.js (~35 KB) загружается асинхронно (флаг `async` в коде счётчика).
  - Рекомендуется отложенная инициализация через `requestIdleCallback` или событие `load` для минимизации влияния на LCP.
  - Ресурсы Вебвизора (~150 KB) загружаются lazy при инициализации.

- **Compliance:**
  - Яндекс Метрика устанавливает cookies: `_ym_uid` (1 год), `_ym_d` (1 год), `_ym_isad` (1 день).
  - При использовании Яндекс Метрики необходимо: (1) добавить уведомление о cookies на сайте; (2) указать использование Метрики в политике конфиденциальности / странице "О сайте".

### Интеграция: Яндекс.Вебмастер `[v1.1 — не в scope v1]`

- **Назначение:** Подтверждение владения сайтом для Яндекса; управление индексацией; мониторинг поисковых запросов.
- **Аутентификация:** Мета-тег верификации в `<head>`:
  ```html
  <meta name="yandex-verification" content="XXXXXXXXXXXXXXXX" />
  ```
  Значение `content` получить в панели Яндекс.Вебмастера после добавления сайта.
- **Действия после верификации:**
  1. "Индексирование" → "Файлы Sitemap": добавить URL `https://<domain>/sitemap-index.xml`.
  2. "Индексирование" → "Переобход страниц": запрашивать после каждой новой публикации.
  3. "Поисковые запросы" → "Популярные запросы": мониторинг позиций.
  4. "Диагностика" → "Безопасность и нарушения": мониторинг проблем.
- **Обработка ошибок:** Если сайт недоступен в момент запроса Яндекс-бота — бот повторит запрос через некоторое время. Sitemap хранится на стороне сайта и обновляется при каждой сборке.
- **Фолбэк:** При отсутствии верификации сайт индексируется Яндексом штатно, но без расширенных возможностей управления индексацией.

### Интеграция: Shiki (подсветка синтаксиса)

- **Назначение:** Синтаксическая подсветка кода в Markdown/MDX-файлах на этапе сборки.
- **Версия:** Встроен в Astro (Shiki 1.x при Astro 5.x).
- **Аутентификация:** Не требуется (локальная библиотека, npm-пакет).
- **Конфигурация в `astro.config.mjs`:**
  ```javascript
  export default defineConfig({
    markdown: {
      shikiConfig: {
        theme: 'github-light', // или кастомная монохромная тема
        langs: ['sql', 'python', 'javascript', 'typescript', 'yaml', 'bash', 'json'],
        wrap: false,   // горизонтальная прокрутка вместо переноса строк
      }
    }
  });
  ```
- **Поддерживаемые языки:**

| Идентификатор | Язык | Примечание |
|---------------|------|------------|
| `sql` | SQL | Запросы к PostgreSQL, 1С Аналитика |
| `python` | Python | RAG-пайплайны, LLM-агенты |
| `javascript` / `js` | JavaScript | Общее |
| `typescript` / `ts` | TypeScript | Конфигурации |
| `yaml` | YAML | Airflow DAG конфигурации, GitHub Actions |
| `bash` / `sh` | Bash | Команды CLI |
| `json` | JSON | Конфигурационные файлы |

- **Вывод:** Shiki генерирует HTML с inline CSS-стилями (не внешние CSS-файлы). Нулевой JavaScript на клиенте.
- **Обработка ошибок:**
  - Неизвестный язык в code fence: Shiki отрендерит блок без подсветки (plain text). Не вызывает ошибку сборки.
  - Синтаксическая ошибка в коде: подсветка применяется «как есть», без валидации корректности кода.
- **Фолбэк:** При отсутствии указания языка (plain code fence ` ``` ` без идентификатора) — блок рендерится без подсветки, только `font-family: monospace`.

### Интеграция: @astrojs/sitemap

- **Назначение:** Автоматическая генерация `sitemap-index.xml` на основе всех статически сгенерированных маршрутов.
- **Версия:** `@astrojs/sitemap` (актуальная стабильная версия для Astro 5.x).
- **Аутентификация:** Не требуется.
- **Обязательная конфигурация:** В `astro.config.mjs` должен быть указан `site`:
  ```javascript
  export default defineConfig({
    site: 'https://<username>.github.io/<repo>',
    integrations: [sitemap()]
  });
  ```
  Без `site` sitemap не генерируется (интеграция выдаёт предупреждение).
- **Генерируемые файлы:** `dist/sitemap-index.xml` + `dist/sitemap-N.xml`.
- **Автоматически исключаемые маршруты:** Нет (все маршруты попадают в sitemap по умолчанию).
- **Ручное исключение** страниц 404 и rss.xml: используется параметр `filter`:
  ```javascript
  sitemap({
    filter: (page) => !page.includes('/404') && !page.endsWith('/rss.xml')
  })
  ```
- **Обработка ошибок:**
  - Если `site` не указан: предупреждение в консоли, sitemap не генерируется. Сборка не падает.
  - Решение: всегда указывать `site` в `astro.config.mjs`.
- **Фолбэк:** При отсутствии sitemap поисковые системы индексируют страницы через обычный краулинг (медленнее, но функционально).

### Интеграция: @astrojs/rss

- **Назначение:** Генерация RSS 2.0 ленты для подписчиков через RSS-ридеры.
- **Версия:** `@astrojs/rss` (актуальная стабильная версия для Astro 5.x).
- **Аутентификация:** Не требуется.
- **Зависимости:** `SITE_NAME`, `SITE_DESCRIPTION`, `SITE_URL` из `src/consts.ts`.
- **Структура файла `src/pages/rss.xml.ts`:**
  ```typescript
  import rss from '@astrojs/rss';
  import { getCollection } from 'astro:content';
  import { SITE_NAME, SITE_DESCRIPTION, SITE_URL } from '../consts';

  export async function GET(context) {
    const notes = await getCollection('notes');
    return rss({
      title: SITE_NAME,
      description: SITE_DESCRIPTION,
      site: SITE_URL,
      items: notes
        .sort((a, b) => b.data.date.getTime() - a.data.date.getTime())
        .map(note => ({
          title: note.data.title,
          pubDate: note.data.date,
          description: note.data.description ?? '',
          link: `/notes/${note.slug}/`,
        })),
    });
  }
  ```
- **Обработка ошибок:**
  - Если запись не имеет `description`: используется пустая строка (не вызывает ошибку).
  - Некорректная дата: `z.coerce.date()` в Zod-схеме гарантирует валидность дат на этапе сборки.
- **Фолбэк:** При отсутствии RSS-ленты пользователи могут добавить URL сайта в RSS-ридер — часть ридеров автоматически обнаруживает ленту через `<link rel="alternate" type="application/rss+xml">` в `<head>`.
