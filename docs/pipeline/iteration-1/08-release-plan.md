## Бизнес-требования

### До запуска (Pre-Launch)

- [ ] **Контент:** Три существующие записи мигрированы из прототипа в Markdown/MDX, визуально верифицированы
- [ ] **Дизайн-верификация:** Каждая страница визуально сравнена с прототипом `Сайт3.html` -- отклонения задокументированы
- [ ] **SEO-готовность:** Мета-теги, sitemap.xml, robots.txt, Schema.org WebSite и BlogPosting -- настроены и проверены
- [ ] **URL-стратегия зафиксирована:** Транслитерация slug для записей и тегов проверена, URL стабильны (не будут меняться после индексации)
- [ ] **Домен / base path определены:** Решение по `https://<username>.github.io/<repo>/` или собственный домен -- принято и настроено в `astro.config.mjs`
- [ ] **Противоречия из QA-отчета устранены:** CRIT-1 (количество тегов), MAJ-1 (иерархия h1), MAJ-5 (динамический год в NoteCard)
- [ ] **Безопасность CI/CD:** `deploy.yml` содержит минимальные permissions, фиксированные версии Actions, `npm ci`
- [ ] **XSS-вектор через MDX проверен:** Тест на `<script>` в MDX-контенте пройден -- скрипт экранируется
- [ ] **GitHub 2FA:** Включена двухфакторная аутентификация на аккаунте автора
- [ ] **Dependabot:** Файл `.github/dependabot.yml` создан для автоматического обнаружения уязвимостей в npm-зависимостях

### День запуска (Launch Day)

- [ ] **Деплой на GitHub Pages:** Push в `main` запускает GitHub Actions workflow, сайт доступен по HTTPS
- [ ] **Проверка всех маршрутов:** `/`, `/notes/`, `/notes/[slug]/`, `/tags/[tag]/`, `/404` -- отдают корректные HTTP-статусы
- [ ] **Проверка RSS:** `/rss.xml` доступен и содержит все 3 записи
- [ ] **Lighthouse аудит (Production):** Performance >= 90, Accessibility >= 95, SEO >= 95
- [ ] **Проверка мобильной версии:** Сайт корректно отображается на реальном мобильном устройстве (320px - 768px)
- [ ] **Индексация запрошена:** Добавить сайт в Google Search Console и Яндекс.Вебмастер, отправить sitemap
- [ ] **Проверка шрифтов:** Все шрифты загружаются с собственного домена, нет запросов к Google Fonts CDN

## Известные ограничения (Known Limitations) для v1.0

Следующие ограничения приняты осознанно и задокументированы для прозрачности:

1. **Кнопка "НАЗАД" всегда ведет на `/`, а не на предыдущую страницу.** Пользователь теряет контекст фильтрации при возврате из детальной страницы. Решение `history.back()` запланировано на v1.1.

2. **Нет пагинации на странице `/notes/`.** При текущем объеме (3 записи) не требуется. При 50+ записях необходим рефакторинг. Запланировано на v2.0.

3. **Класс `.lined-paper` перенесен в CSS, но не используется в v1.** Бэклог-виджет отложен. Класс остается в коде для совместимости с будущими виджетами.

4. **Аналитика не подключена.** Plausible/Umami и Яндекс Метрика задокументированы, но не интегрированы. Подключение в v1.1.

5. **Нет cookie-баннера.** Не требуется до подключения Яндекс Метрики (v1.1).

6. **Правая колонка главной страницы пуста.** Блоки "Статистика" и "Бэклог" из прототипа отложены до v2. Используется упрощенный layout или placeholder.

7. **Twitter Card мета-теги не реализованы.** Минимальное влияние на трафик русскоязычного технического блога.

8. **NFR-15 "100+ записей без изменений в коде" -- уточнение.** Маршрутизация и Content Collections поддерживают 100+ записей. Производительность рендеринга `/notes/` (без пагинации) может деградировать при 50+ записях.

## Решение Go/No-Go

### Текущий статус: УСЛОВНЫЙ GO

**Рекомендация:** **GO** -- при условии устранения критических проблем из QA-отчета до деплоя.

**Обоснование:**

1. **Архитектура проработана.** Семь артефактов покрывают полный цикл от анализа до тестовых спецификаций. Качество документации высокое.

2. **Scope четко определен.** 13 MUST stories + 6 SHOULD stories = минимальный жизнеспособный продукт. Все COULD-фичи осознанно отложены.
