Tech Knowledge Base (Строгий Цифровой Сад)

Tech Knowledge Base — это легковесное одностраничное веб-приложение (SPA), спроектированное для публичной публикации технических заметок, архитектурных решений, фрагментов кода и результатов экспериментов.

Проект объединяет концепцию «Цифрового сада» (Digital Garden) со строгой эстетикой печатных инженерных журналов и дашбордов.

🎯 Концепция и Философия

В отличие от традиционных уютных личных блогов, этот проект транслирует «машинность», строгость и утилитарность. Интерфейс вдохновлен чертежами, терминалами и минималистичными бумажными планировщиками (Bullet Journal в его самой строгой форме).

Ключевые принципы:

Тотальный монохром: Только черный и белый цвета. Идеальная контрастность.

Никакого визуального шума: Полное отсутствие теней (box-shadow), скругленных углов (border-radius) и градиентов.

Геометрия и сетка: Жесткая точечная сетка на фоне, к которой визуально привязаны все элементы. Четкие рамки толщиной 2px.

Информационная плотность: Максимум полезной информации на экране без ущерба для читаемости.

🛠 Технический стек

Проект намеренно не использует тяжелые фреймворки (вроде React или Vue) для обеспечения максимальной скорости загрузки и простоты интеграции с любыми статическими генераторами сайтов (SSG).

Ядро: HTML5, CSS3, Vanilla JavaScript (ES6+).

Стилизация: Tailwind CSS (подключен через CDN, легко переносится в процесс сборки).

Типографика: Google Fonts (Geist, Geist Mono, Unbounded).

Иконки: Lucide Icons (легковесные векторные иконки).

Визуализация данных: Chart.js (используется для рендеринга статистики на Canvas).

🎨 Дизайн-система (UI/UX)

Для поддержания единого стиля при добавлении новых компонентов необходимо соблюдать следующие правила:

Типографика

Заголовки (font-display): Unbounded. Используется только в верхнем регистре (uppercase), с жирным начертанием (font-black, font-bold) для придания брутальности.

Основной текст (font-sans): Geist. Обеспечивает идеальную читаемость лонгридов и интерфейсных элементов.

Технический текст (font-mono): Geist Mono. Используется для дат, тегов, метаданных, кусков кода и навигационных номеров.

UI Элементы

Контейнеры (Боксы): Класс .bujo-box. Строгая рамка border: 2px solid #000000;, белый фон bg-white/95.

Фон: Класс body. Белый цвет с паттерном radial-gradient (точки 1.5px черного/серого цвета с шагом 24px).

Разделители: Только сплошные черные линии (border-b-2 border-black).

Интерактивность: При наведении (hover) элементы часто инвертируют цвет (черный фон, белый текст) или получают жирное подчеркивание с отступом (underline-offset-4 decoration-2).

⚙️ Архитектура приложения

Приложение построено как SPA с собственным минималистичным роутером на Vanilla JS.

Структура данных (Mock Database)

Данные хранятся в объекте db.notes. В реальном проекте этот объект может генерироваться на этапе сборки (из Markdown файлов) или подтягиваться через API.
Обязательные поля: id, title, date, category, tags (массив), type, content (HTML-строка).
Опциональные поля: metrics (для рендеринга специфичных графиков или виджетов).

Роутинг (Объект router)

Роутер управляет состоянием приложения и перерисовкой контейнера #app-content.
Доступные маршруты (Views):

home (Overview / Дашборд): Главная страница со статистикой, последними записями и бэклогом.

all-notes (All Logs): Линейный список всех существующих записей.

tag (Topic): Фильтрация списка записей по выбранному тегу.

detail (Log Entry): Детальная страница конкретной записи с полным контентом.

Переходы между страницами сопровождаются легкой анимацией прозрачности (opacity).

🚀 План дальнейшего развития (Roadmap)

Для трансформации этого прототипа в Production-ready продукт планируется:

Интеграция SSG: Перевод контента из переменной db.notes в Markdown/MDX файлы (использование Astro, 11ty или Hugo).

Синтаксическая подсветка: Добавление библиотеки Prism.js или Highlight.js для красивого рендеринга сложных кусков кода внутри <pre><code>.

Полнотекстовый поиск: Интеграция легковесного поиска (например, Fuse.js) по контенту заметок.

Темная тема: Создание инвертированной версии интерфейса (полностью черный фон, белые линии сетки, зеленый/оранжевый моноширинный текст а-ля ретро-терминал).
